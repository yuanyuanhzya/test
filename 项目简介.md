## 项目简介

本项目是一个完整的电商网站核心模块，聚焦用户认证与前端展示功能，实现了用户注册、登录、注销、权限过滤等核心功能，同时包含首页展示、响应式布局等前端交互场景。采用 Java Servlet 作为后端技术栈，结合 MySQL 数据库存储用户信息，前端使用 HTML、CSS、JavaScript 实现页面交互，整体架构清晰，功能完整。

## 功能模块

### 1. 用户认证模块（核心）

#### （1）登录功能

- **前端**：
  - 登录页面（`login.jsp`）：包含用户名 / 密码输入框、"记住我" 选项、登录按钮、注册链接及第三方登录入口（微信、QQ、微博）。
  - 前端验证（`login_script.js`）：验证用户名 / 密码非空、密码长度≥6 位，实时显示错误提示。
  - 样式（`login_style.css`）：白色背景卡片式布局，橙色主题按钮，响应式适配，hover 动画效果。
- **后端**：
  - `LoginServlet.java`：处理 GET（显示登录页）和 POST（验证登录）请求，已登录用户自动跳转首页。
  - 密码加密：使用`MD5Util`对密码加密后与数据库比对，避免明文存储。
  - Session 管理：登录成功后创建 Session，存储用户信息；"记住我" 选项设置 Session 有效期为 7 天（默认 30 分钟）。
  - 登录状态校验：通过`LoginFilter`拦截未登录用户，自动跳转至登录页（排除登录页、静态资源等白名单路径）。

#### （2）注册功能

- **前端**：
  - 注册页面（`register.jsp`）：包含用户名、密码、确认密码、手机号输入框及注册按钮。
  - 前端验证（`register_script.js`）：验证用户名格式（3-20 位字母 / 数字 / 下划线）、手机号格式（11 位有效号码）、密码长度（6-20 位）、两次密码一致性。
  - 样式（`register_style.css`）：与登录页风格统一，包含格式正确 / 错误的状态提示。
- **后端**：
  - `RegisterServlet.java`：处理 GET（显示注册页）和 POST（提交注册）请求，调用`UserDao`完成用户信息入库。
  - 数据校验：通过`UserDao`检查用户名和手机号是否已存在（避免重复注册）。
  - 注册结果反馈：成功则跳转登录页，失败则提示重试。

#### （3）退出功能

- `LogoutServlet.java`：销毁当前用户 Session，清除登录状态，重定向至登录页。

### 2. 前端展示模块

#### （1）首页布局

- **顶部导航栏**：包含用户登录 / 注册入口、已登录用户信息展示、快捷链接（通过`index_style.css`实现 flex 布局和响应式适配）。
- **Logo 与搜索区**：左侧品牌 Logo，中间搜索框（带橙色渐变按钮），支持关键词搜索。
- **轮播图**：自动播放 + 手动切换（上一张 / 下一张按钮 + 指示器），通过`index_script.js`实现动画逻辑，`index_style.css`控制样式。
- **页脚**：包含网站链接、版权信息，多列 flex 布局展示。
- **回到顶部按钮**：滚动超过 300px 时显示，点击平滑滚动至顶部（带 hover 动画效果）。

#### （2）响应式设计

- 通过`index_style.css`的媒体查询适配不同屏幕尺寸（≤1190px、≤992px、≤768px），调整布局结构和元素大小。

### 3. 数据访问与工具模块

- **数据库交互**：`DBUtil`提供数据库连接与关闭功能，`UserDao`封装用户相关 SQL 操作（登录验证、注册入库、用户名 / 手机号唯一性检查）。
- **实体类**：`User`映射数据库用户表字段（id、username、password、phone、registerTime）。
- **安全工具**：`MD5Util`对用户密码进行加密处理，避免明文存储风险。

## 技术栈

- **后端**：Java 17、Servlet 5.0、MySQL 8.0.33
- **前端**：HTML5、CSS3（Flex/Grid 布局、动画过渡）、JavaScript（DOM 操作、表单验证）
- **构建工具**：Maven 3.9.9
- **服务器**：Tomcat 10.1.19
- **开发工具**：IntelliJ IDEA

## 环境配置

1. **JDK**：版本 17（编译兼容 1.8）
2. **MySQL**：创建数据库及用户表（字段参考`User`实体类），配置`DBUtil`中的连接信息（URL、用户名、密码）。
3. **Maven**：依赖自动下载（Servlet API、MySQL 驱动等）。
4. **Tomcat**：部署项目，上下文路径设为`/`，端口默认 8080。
